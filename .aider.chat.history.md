**Title:** Trendy — Web3 Mini Social App with TRND Rewards

**Description:**
Trendy is a simple Web3-powered social mini app built on Base Network. Users log in with their wallet, create posts, like posts, comment, share content, and earn TRND tokens as rewards. The TRND token has a total supply of 1,000,000,000 (1 billion). The owner holds 300,000,000 TRND, and the remaining 700,000,000 TRND are reserved as user rewards distributed through the Trendy app.

---

## **Core Requirements (Simple Version)**

---

### **1. Wallet Login**

* Enable wallet login for Base & Ethereum networks.
* Users must connect their wallet before posting, liking, commenting, or sharing.

---

### **2. Home Feed (Social Posting)**

Display a scrolling feed of user posts sorted by newest first.
Each post must show:

* wallet address (shortened)
* text (max 200 characters)
* optional image upload
* like count
* comment count
* share count
* timestamp

---

### **3. Create Post**

Form fields:

* text (200 characters max)
* optional image

After submitting:

* Add the post to the feed
* Reward the user **1 TRND** from the reward pool (700M supply)

---

### **4. Like System**

* Users can like any post
* Limit: 1 like per user per post
* When a post receives a like:

  * Post owner receives **0.1 TRND** reward
  * Deduct this from the 700M reward pool

---

### **5. Comment System**

* Users can comment on any post

* Comment fields:

  * wallet address
  * text (max 120 characters)
  * timestamp

* When a post receives a comment:

  * Post owner earns **0.2 TRND** reward
  * Deduct from the 700M reward pool

Comments appear below each post on a modal or expanded view.

---

### **6. Share System**

User can click “Share” on a post.
Sharing creates:

* A re-post that includes:

  * Original post preview
  * “Shared by: [wallet]”
  * New timestamp
* Count share +1

Reward:

* Post owner earns **0.5 TRND** per share
* Deduct from reward pool

---

### **7. Profile Page**

Profile must show:

* connected wallet address
* total posts
* total likes received
* total comments received
* total shares received
* total TRND earned
* "Withdraw TRND" button

---

### **8. Withdraw TRND**

* Implement a withdraw system that sends TRND from the reward supply wallet to the user’s wallet
* Only send what the user has earned
* Ensure withdraw uses Base chain (low gas)

---

### **9. TRND Token Deployment**

Deploy TRND as an ERC-20 token on Base with:

* **Name:** Trendy Token
* **Symbol:** TRND
* **Total Supply:** 1,000,000,000 TRND
* **Owner allocation:** 300,000,000 TRND
* **Reward pool:** 700,000,000 TRND
* Reward pool must be used for:

  * posting rewards
  * likes
  * comments
  * shares

---

### **10. Reward Logic Summary**

Reward distribution from 700M pool:

* **Post:** 1 TRND
* **Like:** 0.1 TRND
* **Comment:** 0.2 TRND
* **Share:** 0.5 TRND

Stop distribution automatically when reward pool runs out.

---

### **11. UI Layout (Simple & Clean)**

Pages:

1. **Home Feed**
2. **Create Post**
3. **Profile**

Every post must include buttons:

* Like
* Comment
* Share

---

* Keep it lightweight and fast to meet Mini App Factory approval standards.
* No need for complex media features.
* No need for messaging or DMs.

  * posting

  * reward distribution
